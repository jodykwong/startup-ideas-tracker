🚫GitHub Pages 的技术限制

静态文件托管

只能托管 HTML、CSS、JavaScript、图片等静态文件，无法运行服务器端代码（Node.js、Python、PHP 等）

无 API 端点支持

不能创建自定义 API 路由，无法处理 POST、PUT、DELETE 等 HTTP 方法，无法进行服务器端数据验证

环境变量限制

无法安全存储 API 密钥，所有配置都会暴露在客户端代码中

构建限制

默认只支持 Jekyll，可以上传预构建的静态文件，但无法运行构建时的服务器端逻辑

✅可行的修改

Row Level Security (RLS) 保护

Gemini API 密钥暴露

复杂业务逻辑

  • 无法实现复杂的业务规则
  • 无法进行安全的第三方 API 调用

🎯推荐部署方案

🏆 最佳方案：GitHub Actions + Vercel

结合 GitHub 的版本控制优势和 Vercel 的全栈支持，既满足您对 GitHub 的偏好，又保持了完整的功能。

📋实施步骤

  1. 设置 GitHub 仓库
    将代码推送到 GitHub，配置 Actions 工作流
  2. 配置 Vercel 集成
    连接 GitHub 仓库到 Vercel，设置自动部署
  3. 环境变量配置
    在 Vercel 中安全地配置 API 密钥和环境变量
  4. 测试部署
    验证所有功能在生产环境中正常工作
  5. 域名配置
    可选：配置自定义域名

GitHub (代码仓库) → GitHub Actions (CI/CD) → Vercel (部署) → 用户访问


数据流: 前端 ↔ Vercel API 函数 ↔ Supabase + Gemini AI
